### Asset licensing policy

- Vector assets: All `.svg` files under `satellites/` and `other-spacecraft/` are original works created by the maintainers and are licensed under CC BY 4.0, the same as other original materials in this repository.
- Raster assets: All committed `.png` files are renders of this repository's own SVGs (regenerated with `tools/render_pngs.mjs`) and are therefore also CC BY 4.0. Earlier revisions of this repository contained rasters resized from third-party websites; those were replaced in July 2026 and are not licensed by this repository — any use of them must comply with the original rights holder's terms.
- Sourced photographs: real satellite photos/renders will be added only where the owner has granted an open licence (public domain, CC BY, CC BY-SA, OGL, agency media terms permitting reuse), with the licence and credit recorded per image in `index.json` (`imageLicense`, `imageCredit`) before the file is committed. An operator's published press/media-page terms permitting use with credit also qualify — recorded as `imageStatus: media-terms` with the terms page as `imageSourceURL` (see the Tier D ladder in `TASKING.md`). Explicit written permission from the owner (e.g. an email reply) likewise qualifies — note sender and date in `ATTRIBUTIONS.csv`. **Attribution is not a licence** — recording where an image came from does not create permission to host it. Never source images from general web/image search.
- The `imageSourceURL` / `imageRightsHolder` / `imageSourceTier` / `imageStatus` fields in `index.json` describe the *photo-sourcing lead* for each mission (where a licensed photo is expected to come from and the state of acquiring it), not the committed files. Committed files are all repo-original until an entry's `imageStatus` records a confirmed licence.
- See `ATTRIBUTIONS.csv` for per-file provenance (source URL, rights holder, stated license/terms). Where an `.svg` is original, the rights holder can be listed as the repository maintainers and the license as CC BY 4.0.

If you are a rights holder and want an asset removed or corrected, please open an issue or contact the maintainer.
